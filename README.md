# Wallpaper API — Static Repository

This repository acts as a **static wallpaper API** for the Indigo Virtual wallpaper app. Wallpapers and metadata are stored here; JSON files are generated by script and consumed via **jsDelivr CDN**.

---

## Repository structure

```
repo-root/
├── wallpapers/          # Full-resolution images (.jpg, .jpeg, .png, .webp)
├── thumbnails/          # Auto-generated thumbnails (400px width)
├── data/                # Generated API JSON files
│   ├── index.json       # API index (totals, page list, latest ref)
│   ├── latest.json      # Newest 20 wallpapers (homepage feed)
│   ├── page-1.json      # Paginated wallpapers (50 per page)
│   ├── page-2.json
│   └── wallpaper-meta.json  # Persisted metadata (filename, author, author_id, source, added)
├── scripts/
│   └── generate-wallpapers-json.js
├── package.json
└── WALLPAPER_API.md     # This file
```

---

## CDN base URL

All asset URLs use jsDelivr:

```
https://cdn.jsdelivr.net/gh/{REPO_OWNER}/{REPO_NAME}@{BRANCH}/
```

Example:

- Image: `https://cdn.jsdelivr.net/gh/username/wallpaper-api@main/wallpapers/sunset-lake.jpg`
- Thumbnail: `https://cdn.jsdelivr.net/gh/username/wallpaper-api@main/thumbnails/sunset-lake.jpg`

Configure in `scripts/generate-wallpapers-json.js`:

```js
const REPO_OWNER = "indigo-virtual";
const REPO_NAME = "indigo-virtual-wallpapers";
const BRANCH = "main";
```

Replace with your GitHub username, repo name, and branch (e.g. `main`).

## Running the generator

**Prerequisites:** Node.js 18+, npm.

1. Install dependencies:

   ```bash
   npm install
   ```

2. Run the script:
   ```bash
   node scripts/generate-wallpapers-json.js
   ```
   Or use the npm script:
   ```bash
   npm run generate:wallpapers
   ```

The script will:

- Scan `wallpapers/` for images (`.jpg`, `.jpeg`, `.png`, `.webp`)
- Generate thumbnails in `thumbnails/` (400px width, aspect ratio preserved) when missing or when the source image is newer
- Persist and merge metadata in `data/wallpaper-meta.json` (filename, author, author_id, source, added)
- Remove metadata and thumbnails for deleted wallpapers
- Write `data/index.json`, `data/latest.json`, and `data/page-1.json`, `page-2.json`, …

Safe to run repeatedly; files are only written when content changes.

## API files (consumed via jsDelivr)

Use the same CDN base; the path is `data/<file>.json`.

| File               | Purpose                                                                        |
| ------------------ | ------------------------------------------------------------------------------ |
| `data/index.json`  | Index: total count, page count, page size, list of page files, latest feed ref |
| `data/latest.json` | Newest 20 wallpapers (for homepage / “latest” feed)                            |
| `data/page-N.json` | Paginated list of wallpapers (50 per page, newest first)                       |

**Example index URL:**

```
https://cdn.jsdelivr.net/gh/username/wallpaper-api@main/data/index.json
```

**Example index.json shape:**

```json
{
  "total_wallpapers": 320,
  "total_pages": 7,
  "page_size": 50,
  "latest": "latest.json",
  "pages": ["page-1.json", "page-2.json", "page-3.json", ...]
}
```

**Example page / latest wallpaper entry:**

```json
{
  "id": "sunset-lake",
  "image": "https://cdn.jsdelivr.net/gh/username/wallpaper-api@main/wallpapers/sunset-lake.jpg",
  "thumbnail": "https://cdn.jsdelivr.net/gh/username/wallpaper-api@main/thumbnails/sunset-lake.jpg",
  "author": "Unknown",
  "author_id": "",
  "resolution": "1920x1080",
  "source": "",
  "added": "2026-03-08T12:00:00Z"
}
```

- **id** — Deterministic slug from filename (e.g. `Sunset Lake.jpg` → `sunset-lake`).
- **image** — Full CDN URL of the wallpaper.
- **thumbnail** — Full CDN URL of the thumbnail.
- **author** — Default `"Unknown"`; override in `data/wallpaper-meta.json`.
- **author_id** — Optional identifier for the author; set in `data/wallpaper-meta.json` (default `""`).
- **resolution** — Default `"1920x1080"`; override in metadata if needed.
- **source** — Optional attribution URL; set in metadata.
- **added** — ISO 8601 timestamp when the wallpaper was first seen; preserved across runs.

## Adding wallpapers

1. Add image files into `wallpapers/` (supported: `.jpg`, `.jpeg`, `.png`, `.webp`).
2. Run:
   ```bash
   npm run generate:wallpapers
   ```
3. Commit `wallpapers/`, `thumbnails/`, and `data/` (and optionally `data/wallpaper-meta.json` if you edited it).

New wallpapers get a deterministic `id` from the filename and an `added` timestamp. Thumbnails are generated automatically.

---

## Editing metadata (author, author_id, source)

The script **never overwrites** `author`, `author_id`, `source`, or `added` with defaults once they are stored. Stored values live in `data/wallpaper-meta.json`.

### How do I set the author name?

You set the author (and optional `author_id`) in **`data/wallpaper-meta.json`**. The script never overwrites `author`, `author_id`, `source`, or `added` once they’re stored there.

**Steps:**

1. **Generate once** so the metadata file exists:

   ```bash
   npm run generate:wallpapers
   ```

2. **Open** `data/wallpaper-meta.json`. You’ll see one object per wallpaper, keyed by **id** (slug from the filename), e.g.:

   ```json
   {
     "sunset-lake": {
       "filename": "Sunset Lake.jpg",
       "author": "Unknown",
       "author_id": "",
       "source": "",
       "added": "2026-03-08T12:00:00Z",
       "resolution": "1920x1080"
     }
   }
   ```

   Each entry includes **filename** (the file name on disk) so you can match wallpapers when you add many at once.

3. **Change** `author` and/or `author_id` (and optionally `source`) for that wallpaper:

   ```json
   {
     "sunset-lake": {
       "filename": "Sunset Lake.jpg",
       "author": "Jane Doe",
       "author_id": "jane-doe",
       "source": "https://unsplash.com/...",
       "added": "2026-03-08T12:00:00Z",
       "resolution": "1920x1080"
     }
   }
   ```

   Don’t remove or change `added`; the script uses it for ordering.

4. **Run the generator again** so the API JSON files get the new author:
   ```bash
   npm run generate:wallpapers
   ```

After that, `latest.json` and the page files will show your author name. In short: add the image → run script → edit `data/wallpaper-meta.json` (set `author`) → run script again.

---

To set or change author, author_id, source, or resolution in general:

1. Run the generator once so `data/wallpaper-meta.json` exists.
2. Open `data/wallpaper-meta.json` and find the entry by wallpaper `id` (slug from filename).
3. Edit `author`, `author_id`, `source`, or `resolution` as needed. Do not remove `added`.
4. Run the generator again so the updated values are written into `latest.json` and `page-*.json`.

Example:

```json
{
  "sunset-lake": {
    "filename": "Sunset Lake.jpg",
    "author": "Jane Doe",
    "author_id": "jane-doe",
    "source": "https://example.com/photo",
    "added": "2026-03-08T12:00:00Z",
    "resolution": "1920x1080"
  }
}
```

The **filename** field is set automatically from the file on disk and helps you identify which entry is which when you add multiple wallpapers at once. Do not remove it when editing.

---

## Removing wallpapers

1. Delete the image from `wallpapers/`.
2. Run:
   ```bash
   npm run generate:wallpapers
   ```
   The script will remove that wallpaper’s entry from all generated JSON and delete its file in `thumbnails/`.

---

## Pagination and ordering

- Wallpapers are ordered **newest first** by `added`.
- **Page size:** 50 wallpapers per page.
- **latest.json:** Contains the 20 newest wallpapers for a fast “latest” feed without pagination.

## Using this API from the app (private repo)

In the app repo, load the API via jsDelivr using your real repo owner, name, and branch:

1. **Index:**  
   `GET https://cdn.jsdelivr.net/gh/{owner}/{repo}@{branch}/data/index.json`  
   → Use `total_wallpapers`, `total_pages`, `pages`, `latest`.

2. **Latest (homepage):**  
   `GET https://cdn.jsdelivr.net/gh/{owner}/{repo}@{branch}/data/latest.json`  
   → Use `wallpapers` array.

3. **Page N:**  
   `GET https://cdn.jsdelivr.net/gh/{owner}/{repo}@{branch}/data/page-{N}.json`  
   → Use `wallpapers` array.

All image and thumbnail URLs in the JSON are already full jsDelivr URLs; use them as-is.

---

## Uploading to GitHub

1. Set `REPO_OWNER`, `REPO_NAME`, and `BRANCH` in `scripts/generate-wallpapers-json.js` to match your repo.
2. Run `npm run generate:wallpapers` so `data/` and `thumbnails/` are up to date.
3. Commit and push. jsDelivr will serve the repo at:
   `https://cdn.jsdelivr.net/gh/{REPO_OWNER}/{REPO_NAME}@{BRANCH}/`
4. Optionally add a GitHub Action to run the script on push (e.g. when `wallpapers/` or `scripts/` change) so `data/` and `thumbnails/` stay in sync.
